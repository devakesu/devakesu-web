## ðŸ”’ Security Measures

### Application Security

- **SLSA Level 3 Provenance** - Verifiable build integrity
- **Trivy Scanning** - Automated vulnerability detection in CI/CD
- **Security Headers**:
  - `X-Frame-Options: SAMEORIGIN` - Clickjacking protection
  - `X-Content-Type-Options: nosniff` - MIME sniffing prevention
  - `Referrer-Policy: origin-when-cross-origin` - Privacy control
  - `Permissions-Policy` - Feature restrictions
- **React Strict Mode** - Development safety checks
- **No Powered-By Header** - Reduced information disclosure

### Code Security

- All external links use `rel="noopener noreferrer"`
- Scripts loaded via Next.js Script component (CSP-friendly)
- No inline scripts or eval()
- Try-catch error boundaries in all JS modules
- Null/undefined checks with optional chaining

### Privacy

- âœ… No tracking scripts or analytics
- âœ… No cookies or local storage
- âš ï¸ Minimal third-party request: Background noise SVG loaded from `https://grainy-gradients.vercel.app/noise.svg` (receives IP address and user-agent like any HTTP request)
- âœ… GDPR compliant by design
- âœ… Client-side only architecture

## ðŸŽ¨ Customization

```css

```

### Fonts

Fonts are loaded in `app/layout.js`:

Edit `app/page.js` to update:

- Personal information
- Projects showcase
- Skills and tech stack
- Contact information

### Build Metadata

The terminal displays real build data from `/public/meta.json`, which is automatically generated during the build process via the `prebuild` script. In production (Coolify), it uses environment variables from the git repository. In CI, it includes audit and provenance status.

## ðŸš¢ Deployment

### Automated CI/CD Pipeline

The project uses GitHub Actions with a secure, multi-stage pipeline:

1. **Security Audit** - Trivy vulnerability scanning
2. **Build** - Next.js build with metadata injection
3. **SLSA Provenance** - Cryptographic build verification
4. **Deploy** - Coolify deployment via webhook

See [.github/workflows/deploy.yml](.github/workflows/deploy.yml) for details.

### Required Secrets

Add these to your GitHub repository secrets:

- `COOLIFY_BASE_URL` - Your Coolify instance URL
- `COOLIFY_APP_ID` - Application UUID
- `COOLIFY_API_TOKEN` - API authentication token

### Manual Deployment

```bash
# Build the project
npm run build

# Deploy the .next/standalone or dist folder to your hosting provider
```
